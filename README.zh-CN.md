# 🌊 Tide（潮汐）

[English](./README.md) | **中文**

> 资讯如潮汐，一波接一波，永不停歇。你无法阻挡海洋，但你可以学会读懂它。

**Tide** 是一个开源的个人信息摄入引擎：订阅对你真正重要的信息源（arXiv 论文、Twitter/X 上的声音、AI 实验室博客……），但它不满足于做摘要，而是产出**深度解读**，并汇入你的个人知识库——让你消费的信息真正变成你拥有的理解。

## 为什么叫 "Tide"（潮汐）？

新闻、论文、热点每天像潮水一样涌来。大多数聚合工具试图*把海喝干*：抓取一切、压缩成要点、完事。结果是一条你刷完就忘的信息流。

Tide 站在相反的立场上。潮水无论如何都会来——问题不是"我怎么看完所有东西"，而是"**我允许哪些东西塑造我的思考，以及塑造到多深？**"。Tide 替你观察水面，挑出值得研究的浪，帮你认真地与它们交手。

## 核心信念：理解是无法外包的

这个项目建立在一个想法之上，来自 Andrej Karpathy：

> **"理解是无法外包的。"（Understanding cannot be outsourced.）**

在 AI 时代，把*阅读*外包出去从未如此容易。任何一个 LLM 都乐意把一篇论文压缩成五个要点。但一份你没有与之搏斗过的摘要，就不是你真正拥有的东西。如果消化全部交给 AI，你最终得到的只是一座整理得井井有条、却从未真正理解过的图书馆。

所以 Tide 刻意**不**以"帮你省去阅读"为目标。它的目标是：

1. **过滤** —— 呈现真正值得你关注的少数内容，而不是每天倾倒 50 条。
2. **深度解读** —— 对每一条入选内容做真正的分析：它在攻克什么问题、真正的新意在哪、它与你已有的认知如何关联、它可能错在哪里。
3. **汇入知识库** —— 输出的目的是*融入*个人 wiki（Karpathy 式的 LLM 维护知识库），而不是看完即弃。
4. **把理解留给你** —— Tide 负责备好材料、提出问题；最后那一步理解，始终由人完成。

把它想象成一个替你预读的研究助理——而不是替你阅读的替身。

## Tide 做什么

- 📡 **订阅** —— arXiv 主题/关键词、精选的 Twitter/X 账号（研究者、builder、实验室）、博客等一切可 RSS 化的信息源。
- 🌊 **采集与去重** —— 每日抓取、跨源去重、噪声清理。采集是脏活，它应该无聊而可靠。
- 🔍 **打分与精选** —— 基于*你自己*声明的兴趣做相关性打分，让每日精选既少又个人化。
- 🧠 **深度解读** —— 多步 LLM 分析链（不是一次性摘要）：背景 → 贡献 → 批判 → 与你已有笔记的关联。
- 📚 **知识库输出** —— 为 Obsidian 式个人 wiki 设计的 Markdown 输出，带指向相关概念的双向链接。
